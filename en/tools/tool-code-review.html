Dev Support / Code Review

Code & Security Review System

One /review command runs 4 checks simultaneously. Three custom slash commands plus the official /security-review cover cost, data, logic, and security risks — without touching your code.

Overview

Coverage

Security · Cost · Data · Logic — 4 dimensions

How to trigger

/review runs all 4 commands at once. Add "full" to switch between diff-only and full scan

Safety guarantee

Read-only by design — AI reports, never modifies

Problem & Solution

Before

The more code AI writes, the harder it is to spot cost explosions, data corruption, and logic bugs. Generic review tools focus on security and miss everything else.

After

Type /review at any checkpoint and get all 4 dimensions checked at once. AI only reports — it never modifies. Use /review for diff-only, /review full for a full scan.

How It Works

01 Type /review at any checkpoint (internally runs /cost-review, /data-review, /logic-review, and /security-review in sequence)
02 AI reports risks across all 4 dimensions — no modifications, no rollbacks
03 Review the report and decide what to fix yourself
04 Use /review full at major milestones for a complete scan; use /review day-to-day for a lightweight diff check

Background

The more AI writes code, the more cost explosions, data corruption, and logic bugs accumulate — and official tools only cover security. So I split the concerns into four independent commands that only fire when you manually invoke them. Bundling them into /review removed the friction of running each one separately. Three principles stayed fixed: no interdependencies, no auto-trigger, no modifications.
