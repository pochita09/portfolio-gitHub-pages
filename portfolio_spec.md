# ポートフォリオサイト 要件定義・設計図

## 1. 概要

| 項目 | 内容 |
|---|---|
| 目的 | AIツール制作（バイブコーディング）の受託・営業用ポートフォリオ |
| 公開先 | GitHub Pages（静的サイト） |
| 制作方法 | Claude Code によるバイブコーディング |
| デモの見せ方 | 動画（表紙はスクショ）。公開できるツールは別途リンク |

---

## 2. 技術要件

- 静的サイトとして成立すること（GitHub Pages で動く。サーバー側処理は使わない）
- 入力フォームは作らない → 問い合わせは mailto または外部フォーム（Googleフォーム等）へのリンク
- スマホ対応（レスポンシブ）
- 凝ったアニメーション・重いライブラリは不要。軽量・高速優先
- ダークモード対応は任意
- 本文・名前・数字は当面プレースホルダー。差し替えやすいよう、テキスト/ツールデータは1か所にまとめて定義する

---

## 3. デザイン方針

- ミニマルで落ち着いた、信頼感のある見た目
- ネオン・3D・過剰なアニメは使わない
- 余白を活かし、ツール（中身）を引き立てる

---

## 4. サイト構造（ページ一覧）

| ページ | 内容 |
|---|---|
| トップ | ヒーロー＋代表ツール3つ＋その他ツール一覧 |
| ツール個別ページ × 5 | 同一の型を使い回す |
| About | 自己紹介 |
| Contact | 問い合わせ導線（mail/ 外部フォームリンク） |

ナビゲーション：Tools / About / Contact の3項目のみ

---

## 5. トップページ設計

上から順に：

1. **ヘッダー** — 屋号/名前、ナビ（Tools / About / Contact）
2. **ヒーロー** — 一行キャッチコピー ＋ 2行の自己紹介 ＋「お問い合わせ」ボタン
3. **代表ツール（3つ）** — 枠で強調したカードを横並び（グリッド）。各カードに：サムネイル画像／カテゴリタグ／ツール名／1行説明。クリックで個別ページへ
4. **その他ツール（2つ）** — 小さめのカードで一覧（グリッド）
5. **フッター** — コピーライト ＋「このサイトもバイブコーディングで作っています」

### ツール一覧（仮名）

| 区分 | ツール名（仮） | カテゴリ（仮） |
|---|---|---|
| 代表1 | 現場画像 → 報告書 自動生成 | 現場 / 報告書 |
| 代表2 | 育休制度 AI相談（RAG） | 人事 / 制度 |
| 代表3 | RSS 重要度分析・要約 | 情報収集 |
| その他1 | Obsidian + Gmail 連携 | 生産性 |
| その他2 | コード品質チェック | 開発支援 |

---

## 6. ツール個別ページ設計（5ページ共通の型）

上から順に：

1. **「Tools一覧へ戻る」リンク**
2. **見出し** — カテゴリタグ ＋ ツール名 ＋ 1行説明
3. **デモ動画エリア** — 表紙にスクショ画像を表示し、再生ボタンを重ねる。クリックで動画再生（動画ファイルは後で差し込めるよう仮枠）
4. **要点カード3つ（横並び）** — ラベル＋値。例：「作成時間：30分→数十秒」「対象：現場系の報告」「入力：写真を添付」※数字は実測値に差し替える前提
5. **「こんな課題を解決します」** — Before → After を文章で
6. **「使い方（3ステップ）」** — 番号付き3項目
7. **「作った背景」** — 短い文章
8. **「使った技術」** — タグ数個（例：画像解析AI／文書生成AI／バイブコーディング）
9. **問い合わせ導線** — 一言 ＋ メールリンク

---

## 7. Contact / About

- **Contact** — フォームは作らない。mailto または外部フォームへのリンク。当面は仮のメールアドレス
- **About** — 簡単な自己紹介ページ

---

## 8. 制作の進め方（段階指定）

1. まず **トップページ ＋ 個別ページ1つ（代表1：現場画像→報告書）** だけを作る
2. 確認して OK が出たら、残りのページへ展開
3. 各ステップでブラウザで見た目を確認できる状態にする

---

## 9. 差し替え対象（後から埋める項目）

- 屋号 / 名前
- ヒーローのキャッチコピー・自己紹介文
- 各ツールの正式名称・カテゴリ
- 各ツールの Before / After 文章・使い方・背景
- 要点カードの実測値（数字）
- サムネイル画像・スクショ・デモ動画
- 問い合わせ先メールアドレス / 外部フォームURL
